# career / learn-this

## Before you start

Read `pack/domains/career/state.md`.

## How to run

Ask: *"What are you trying to understand, and where did it come up?"*

Then:

1. Explain in three or four sentences, plain English.
2. Pause. Check what clicked.
3. Go deeper where asked.
4. Connect to the user's own chapter where relevant.

If the topic is legal (notice periods, non-competes, IP clauses, redundancy law), flag that an employment solicitor matters more than Claude.

## After the conversation

Offer to log as an observation, or add to Open questions if it surfaced one.

## Voice reminder

No lecture. Brief, anchored, practical.
