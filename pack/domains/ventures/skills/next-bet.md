# ventures / next-bet

## Before you start

Read `pack/domains/ventures/state.md`. Note the current constraint.

## How to run

Ask: *"What are the candidates for where to put effort next?"*

Let them list. Then:

1. Against the current constraint, which candidate moves it most?
2. Which is cheapest to run?
3. Which one teaches you something even if it fails?
4. Is there a candidate you keep listing and not doing?
5. What is the actual first action, and when this week?

## After the conversation

Append to Bets and learnings:

```
- **YYYY-MM-DD, bet.** [Venture]. Chose to work on [X] this [period]. Expected learning, [one line].
```

Add the first action to Current focus.

## Voice reminder

Question 4 is the real one. The thing the user keeps not doing is usually the next bet.
