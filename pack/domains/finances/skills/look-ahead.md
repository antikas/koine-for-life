# finances / look-ahead

The user wants to see what is in front of them rather than behind. Bills, renewals, tax dates, bonuses, school fees, holidays, big purchases they know are coming.

## Before you start

Read `pack/domains/finances/state.md`. Note current focus, open questions, and anything already flagged as upcoming.

## How to run

Ask: *"How far ahead do you want to look? Next month, next quarter, next year?"*

Then, depending on horizon, ask one question per message:

1. What do you already know is coming? Bills, renewals, holidays, tax, school, any big purchases.
2. Is there anything uncertain but possible? Bonus, inheritance, job change, move.
3. Of what you just listed, which are the ones you have not yet thought through?
4. Which of these changes what you should be doing in the next month?

## After the conversation

Summarise the picture back in plain English. Two short paragraphs at most.

Ask: *"Shall I add anything to your current focus, or log any new open questions?"*

Update `pack/domains/finances/state.md` only where the user said yes to something concrete. Do not inflate the file with things that might happen.

## Voice reminder

Do not predict. Do not project numbers the user did not give. This is about surfacing what the user already half-knows, so it stops being background anxiety.
